## 背景

当前项目已具备用户、文章、分类管理功能，但缺少个人效率工具。新增任务管理模块，让用户能够追踪待办事项，使应用在内容管理之外更具实用性。同时，整个前端布局从顶部 tab 导航改为侧边栏导航，所有模块统一通过侧边栏访问。

## 变更内容

- **后端**：新增 `Task` 实体、JPA Repository、Service、DTO 和 REST Controller，提供完整 CRUD 接口以及状态切换功能
- **前端**：新增 `TaskManager.vue` 页面组件，路由为 `/tasks`
- **布局改造**：将 `App.vue` 从顶部横向 tab 导航重构为深色侧边栏布局，所有模块（仪表盘、任务管理、用户管理、文章管理、分类管理）统一通过侧边栏访问
- **路由**：Vue Router 新增 `/tasks` 路由，带认证守卫
- **API 层**：前端 API 客户端新增 `taskApi` 模块

## 能力

### 新增能力

- `task-manager`：完整的任务 CRUD 管理——创建、列表、更新（标题、描述、状态）、删除。每个任务包含标题、可选描述、完成状态（已完成/未完成）和时间戳。前端页面通过侧边栏导航访问。
